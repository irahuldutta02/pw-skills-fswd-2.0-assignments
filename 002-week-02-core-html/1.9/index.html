1. Headings (h1-h6)
    • Use headings to create a clear and organized hierarchy of information on a web page.
    • Use only one h1 tag per page, and use the remaining heading levels to create a logical structure.
    • Use descriptive and meaningful text for each heading to improve accessibility and SEO.
  2. Paragraphs (p)
    • Use the p tag to create blocks of text on a web page.
    • Avoid using multiple line breaks or empty paragraphs for formatting purposes.
    • Use CSS to control the spacing and indentation of paragraphs.
  3. Lists (ul, ol, li)
    • Use unordered lists (ul) for lists without a specific order, and ordered lists (ol) for lists with a specific order.
    • Use the li tag to create individual list items.
    • Avoid using lists for formatting purposes, and use CSS for styling instead.
  4. Links (a)
    • Use the a tag to create clickable links to other pages or resources.
    • Include descriptive text within the link tag to improve accessibility and SEO.
    • Use the target attribute to control how the link opens (e.g. in a new window or tab).
  5. Images (img)
    • Use the img tag to embed images on a web page.
    • Include the alt attribute to provide a description of the image for accessibility and SEO.
    • Optimize images for web by reducing file size and using appropriate formats.
  6. Emphasis (em, strong)
    • Use the em tag to indicate emphasis or stress in text.
    • Use the strong tag to indicate importance or significance in text.
    • Avoid using these tags for formatting purposes, and use CSS for styling instead.
  7. Inline code (code)
    • Use the code tag to indicate inline code or programming language syntax.
    • Use preformatted text (pre) for longer blocks of code.
    • Use syntax highlighting to improve readability of code.
  8. Blockquotes (blockquote)
    • Use the blockquote tag to indicate quoted text or citations.
    • Include the cite attribute to provide the source of the quote.
    • Use CSS to control the indentation and styling of blockquotes.
  9. Horizontal rule (hr)
    • Use the hr tag to create a horizontal line to visually separate content.
    • Avoid using multiple hr tags for formatting purposes, and use CSS for styling instead.
    • Include whitespace before and after the hr tag for visual separation from surrounding content.
